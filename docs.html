Vault Ceaser — API & architecture

Paths are under your app origin (host + port). While the server runs, OpenAPI + try-it UI: /docs. This page is also served at /docs.html (same content as docs.html in the repo).
v2: Anikoto.tv is now the primary source for episode lists and streams; Miruro pipe is fallback. HTTP client for anikoto uses wreq (Chrome TLS fingerprint).

General

GET/Service name, version, short index of route names.
GET/healthProcess liveness: status and version.
GET/api/healthUpstream checks: each configured pipe base (HEAD latency), circuit-breaker state, embed_s2_mode, stream_upstream_base, config path.

Configuration (config.json)

Override path with env VAULTCEASER_CONFIG. Restart the server after edits.

miruro_basePrimary origin for /api/secure/pipe (e.g. https://www.miruro.tv).
pipe_fallback_basesJSON array of extra pipe origins, same API contract. Tried in order when the primary fails or is temporarily marked dead.
pipe_dead_ttlSeconds (default 120) a pipe host stays deprioritized after a transport / 5xx / decode failure. Client 4xx from the pipe does not mark the whole host dead.
stream_upstream_baseJWPlayer embed site (e.g. https://vidwish.live) — /stream/s-2/…, /stream/getSources.
embed_s2_modeupstream = real upstream iframe (numeric id resolved from pipe). synthetic / proxy = local player or proxied HTML, more traffic via /api/cdn-hls.
cdn_host_suffixesAllowlist of host suffixes for /api/cdn-hls.
embed_asset_hostsHosts used when rewriting embed asset URLs.
anilist_urlGraphQL endpoint for catalogue metadata.

Show details

{anilist_id} is the numeric AniList media id.

GET/api/anime/{anilist_id}Full metadata plus streaming hints (links to episodes and stream endpoints).
GET/api/anime/{anilist_id}/characterspage, per_page.
GET/api/anime/{anilist_id}/relationsRelated titles.
GET/api/anime/{anilist_id}/recommendationspage, per_page.
GET/api/anime/{anilist_id}/episodesEpisode list — anikoto.tv primary (ids formatted anikoto:{ep_id}), Miruro pipe fallback. Includes original_id used by /api/stream/iframe.
GET/api/anime/{anilist_id}/streamEpisodes with resolved stream info for the first few entries. Query: provider (default megaplay), category (sub · dub · ssub), optional episode_number.

Playback

Stream category: sub, dub, ssub. Provider: megaplay. Pipe uses provider key bee internally for slug ids.

GET/api/sourcesRequired: episode_id, anilist_id. Optional: provider, category. Normalized sources for a player.
GET/api/stream/urlRequired: episode_id. Optional: provider, category, anilist_id. Single HLS URL plus tracks when present.
GET/api/stream/iframeRequired: episode_id. Optional: category, anilist_id. For anikoto:* ids: resolves full anikoto chain → megaplay URL. For slug ids: Miruro pipe. JSON: iframe_url, embed_html, embed_s2_mode (anikoto | upstream | synthetic).
GET/watch/{anilist_id}HTML watch UI (episode list + iframe). Not JSON.

Proxies

GET/api/cdn-hls?u=…&r=…u: percent-encoded absolute URL (m3u8, segment, vtt). r (optional): per-stream Origin/Referer base from pipe entries (e.g. megaplay vs vidwish). GET, HEAD, OPTIONS.
GET/api/mp/…Reverse proxy to the embed host (pages, getSources, assets). Path mirrors upstream under this prefix. GET, HEAD, OPTIONS.

Advanced

POST/api/pipeJSON body: required path; optional method (default GET), query, referer. Pass-through to the configured pipe API.
GET/api/episode-countsBatch sub/dub released counts. Required: ids (comma-separated, max 50). Optional: refresh=true. Source: Miruro pipe (5-min cache).

Anikoto integration (v2)

Episode data and stream URLs are resolved via anikototv.to. HTTP client uses wreq (Chrome 128 TLS fingerprint). Miruro pipe remains as fallback for any anime not found on anikoto.

Cache keyanilist_id → anikoto_id24 h. Resolved via title search + watch page scrape.
Cache keyanikoto_id → episodes[]5 min. Parsed from /ajax/episode/list/{id}?vrf=.
Cache keylink_id → megaplay_url1 h. Resolved via /ajax/server?get={link_id}.
Episode id formatanikoto:{ep_id}Opaque id passed to /api/stream/iframe. Server looks up data_ids blob from in-memory cache to resolve the stream chain.
